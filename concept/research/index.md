# research/, the verified sources

The source research briefs for the framework (with their URLs at the bottom of each file).

- [01, Karpathy & practitioners](01-karpathy-practitioners.md), How Karpathy, Willison, swyx, Litt, and Forte organize their knowledge to work with an LLM. Concrete mechanics and patterns worth borrowing.
- [02, Anthropic / Claude Code architecture](02-anthropic-architecture.md), The canonical building blocks (CLAUDE.md, skills, subagents, hooks, MCP) and the agent-engineering principles Anthropic has published.
- [03, AI agent memory architectures](03-memory-architectures.md), The CoALA taxonomy, temporal memory (Zep/Graphiti), Letta/mem0, and the decision guide for markdown+git versus heavier infrastructure.
- [04, PKM methods in the age of the LLM](04-pkm-methodologies.md), PARA, Zettelkasten, Johnny.Decimal, digital garden, OKF, judged by how readable they are to an LLM agent. The note-taking convention for a markdown+git KB.
- [05, Loops, verification & orchestration](05-loops-orchestration.md), The discover→plan→execute→verify→iterate cycle, open versus closed, single versus fleet, maker≠checker, stopping conditions, and the pragmatic defaults for a solo operator.
- 06, LIVING REFERENCE / CANON FLOTANT (JP Noto): **private source, credited but not published.** The 3 concepts carried over: double value (law L8), traced human validation (Spec §11), status cycle (Spec §6).
- [07, OpenClaw: identity conventions](07-openclaw-identity.md), The agent's identity as versioned markdown files (SOUL.md, USER.md, AGENTS.md…): what Manence borrows (SOUL.md, the spirit) and where it diverges (STRATEGY.md, the human in the Knowledge).
